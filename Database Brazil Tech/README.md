# PROJETO DE BANCO DE DADOS DE E-COMMERCE 

Projeto de banco de dados relacional simulando um E-commerce.

# Solução de Negócio

Este projeto fictício busca resolver um problema comum em empresas de E-commerce: organizar e analisar dados de clientes, pedidos e produtos de forma eficiente.  

A modelagem proposta atende às seguintes necessidades:
- **Gestão de Clientes e Pedidos:** cadastro estruturado de clientes e acompanhamento dos pedidos realizados.
- **Controle de Estoque e Produtos:** organização de informações de produtos, categorias e disponibilidade.
- **Relacionamento N:N entre Produtos e Pedidos:** suporte para que um pedido contenha múltiplos produtos e um produto esteja presente em diversos pedidos.
- **Análises de Vendas:** relatórios com métricas como total de vendas, produtos mais vendidos e comportamento dos clientes.
- **Insights Estratégicos:** apoio a decisões de negócio como promoções, reposição de estoque e fidelização de clientes.

Assim, o banco de dados fornece uma base sólida para que gestores de um E-commerce possam tomar decisões baseadas em dados.

# Estrutura

- **schema.sql** → Modelagem de dados  
- **inserts.sql** → Inserção de dados  
- **analytics.sql** → Consultas analíticas  

# Conceito Aplicado

- Modelagem Relacional  
- Relacionamento N:N  
- JOINS  
- Agregações (SUM, COUNT)  
- Funções de String  
- Manipulação de dados
